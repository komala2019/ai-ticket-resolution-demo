Process Sequence

The flow below shows how a single issue moves through the system, from customer message to routing decision and final feedback.

CUSTOMER
1. Issue described
The customer types a plain-language support request in the chat composer, with optional media attached.
AI AGENT
2. Scoring and matching
The classifier compares the query against the active KB, scenario data, and historical tickets to derive confidence.
ROUTING ENGINE
3. Threshold triage
The score is routed into one of four live bands:
>= 90% conf Auto-resolve
The fix is shown in chat and the ticket closes when the customer confirms it worked.
75-89% conf CS approval queue
A rep reviews the draft response, edits it if needed, and approves it before sending.
50-74% conf CS rewrite queue
The rep performs a deeper review and can rewrite the response before it reaches the customer.
< 50% conf Engineering queue
Novel issues are escalated with diagnostic details, reproduction context, and a P1 handoff.
Feedback loop
FEEDBACK LOOP
4. Closed-loop learning
Thumbs up closes the loop, thumbs down flags the KB article, and CS edits feed back into the shared live state.
