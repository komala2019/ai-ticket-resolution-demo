AI Ticket Auto-Resolution

An end-to-end human-in-the-loop support demo. The shell switches between Customer chat, CS Console, and docs. A shared DemoStateService keeps KB, queue, notifications, and analytics in sync across all views.

Core Concept

The app follows a simple loop: surface known issues upfront, let the customer describe the issue, clarify the context, classify the ticket, route it by confidence, resolve it, and confirm the outcome.

0
Known Issues
Before the customer types anything, active known bugs are displayed upfront as an accordion — one click shows the workaround.
1
Describe
The customer enters the issue in plain language instead of filling out a form.
2
Clarify
The assistant asks targeted follow-up questions to remove ambiguity before making a match.
3
Classify
The live classifier compares the query against the knowledge base and historical tickets to produce a confidence score.
4
Route
Thresholds split the ticket into Auto-resolve, CS approval, CS rewrite, or Eng queue.
5
Resolve
Known fixes are sent directly, known issues get a workaround, and novel cases are escalated with context.
6
Confirm
Customer feedback updates the metrics, flags the KB if needed, and improves future routing.

Resolution Paths

Known Issue upfront Zero-typing self-service

The customer clicks a known bug in the upfront panel and is taken straight to the workaround card. No message needed — the KB match is instant.

Type 3 - Happy Path Auto-resolve in chat

The issue matches a clear KB article, the customer sees the fix immediately, and the ticket closes when they confirm it worked.

Type 2 - Agent Assist CS approval queue

A known bug or partial match surfaces a workaround plus estimated resolution time. A CS draft lands in the approval queue for review before sending.

Type 1 - Escalation Engineering queue

Novel issues are packaged with reproduction steps, environment data, and blast-radius context so engineering can act without re-asking for basics.

Key Features

Each feature below is live in this build and observable during a normal demo walkthrough.

Known issues upfront panel
Zero-typing self-service for KB known-bug entries. Active bugs appear as an accordion before the customer types a single word.
Multi-level chip drill-down
Area chips expand to sub-chips which feed into classification — no clarification loops. Hard cap at two rounds.
Markdown rendering in AI messages
Bold, italic, and bullet syntax in AI replies is rendered safely in chat bubbles via MarkdownPipe and DomSanitizer.
Human / agent handoff
The "Human" button connects to Maya (support specialist), who replies contextually. Auto follow-up fires after placeholder replies.
State persistence across tabs
Chat uses [hidden] not *ngIf so Angular never destroys the component. Conversation state survives view switches to the CS console and back.
Live analytics + notifications
Resolve, reopen, and escalate actions update KPI cards and the topbar bell instantly. Each notification can be dismissed individually.

Closed Loop Learning

The app is designed to show how each action changes the next one. Customer feedback and agent edits update the same live state used by the console and the analytics view.

Thumbs Up
Confirms the fix, closes the ticket, increments the deflection metrics, and increases KB usage for the matching article.
Thumbs Down
Reopens the ticket, lowers accuracy, and flags the matching KB article for review so the instructions can be improved.
CS approval and KB curation
Approved drafts can become better KB entries, and edited articles immediately change how future tickets are matched.

How To Test The Demo

1
Check the known issues panel
Open Customer view and click "Analytics dashboard blank on Chrome 124+" in the known issues accordion. Hit the workaround button to see zero-typing self-service.
2
Try the customer flow
Type "booking widget disappeared" and follow the assistant through the fix steps.
3
Trigger a reopen
Run the same scenario again and click "Still broken" to see KB flagging and the accuracy metric drop live.
4
Use sub-chips
Type "Analytics" or click "Wrong data shown" to see the area chip expand into sub-chips and feed into a classification — the two-level drill-down in action.
5
Review the console
Switch to CS console and inspect the Approval Queue, KB Manager, Analytics (hover card titles for formula tooltips), and Golden Set regression suite.
